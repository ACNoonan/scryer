# scryer ‚Äî Phase log

Running ledger of v0.1 phases. What shipped, when, and why.

This file holds two append-only sections moved out of
`methodology_log.md` (Decision log, Specification log) plus the
"Done ‚Äî shipped in v0.1" index that used to sit at the top of
`wishlist.md`.

- The **Done index** is the user-facing summary keyed by wishlist
  item number; it points at the phase row for full detail. Per the
  2026-05-01 "Done definition ‚Äî code-shipped vs data-shipped" lock
  in `methodology_log.md`, an entry only appears here once **both**
  (a) schema/fetcher/CLI code is merged AND (b) at least one
  persisted parquet partition exists at the canonical dataset root
  for the declared date range. Code-only phases live in the
  "Code-shipped, data-pending" sub-section below.
- The **Decision log** is the architectural-decision audit trail ‚Äî
  one row per phase, with the change and the reason it was done that
  way. Hard rule #1 in `CLAUDE.md` says new phases append a row here.
- The **Specification log** is reserved for cases where multiple
  approaches were tried and benchmarked; empty in v0.1.
